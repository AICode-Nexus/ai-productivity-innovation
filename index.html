Core Thesis

AI 的下一轮红利，不在“多用几个工具”，而在“重写工作方式”。

个人已经能用 AI 写、查、算、画、总结；企业真正的分水岭，是先准备好场景、数据、官方账号、工具入口、国内外算力、人才和治理，再把这些能力嵌入流程，明确输入、输出、质检、责任和复盘。

Trend

国内外 AI 提效发展的共同信号

海外正在从 Copilot 走向 Agent 和组织重构；国内正在从模型能力、行业试点走向场景落地和流程改造。越往 Agent 走，账号归属、地区支持、用量限制、中转链路和推理算力越会变成生产约束。

Emerging Jobs

AI 时代的新兴岗位：从会用工具，到能落地系统

这些岗位不是为了追热词，而是补齐企业把 AI 放进真实流程时最容易缺失的能力。

FDE

Forward Deployed Engineer / 前线部署工程师

解决的问题
AI 能力到业务现场之间的部署断层。
典型产出
现场流程图、数据接入清单、试点方案、上线检查表。
配置时机
已有明确场景，但 IT、数据和一线团队协作成本较高。
Agent

AI Agent 工程师

解决的问题
任务链自动执行、工具调用和人机交接设计。
典型产出
Agent 任务说明、工具权限、异常升级规则、日志和回滚方案。
配置时机
已完成流程梳理，准备把低风险重复任务交给 Agent。
Product

AI 产品工程师

解决的问题
模型能力如何变成真实可用的产品功能。
典型产出
AI 功能 PRD、交互状态、评估样例、灰度计划。
配置时机

Context

上下文/知识工程师

解决的问题
AI 不知道该信什么材料、该引用什么知识。
典型产出

配置时机
文档分散、制度复杂，知识准确性直接影响交付质量。
Governance

AI 评测与治理负责人

解决的问题
AI 输出质量、合规、安全和责任边界无法判断。
典型产出

配置时机
金融、医疗、工业等高风险场景，或 AI 功能准备对外上线。
Enablement

AI Ready Course

企业 AI Ready：上 AI 前先准备这六件事

AI 提效不是先买工具，也不是先堆算力。企业要把业务目标、可信材料、工具入口、官方账号、模型供给、国内算力、人员流程和治理机制放到同一张准备清单里。

Reality Check

当前账号、中转站与国内算力问题

截至 2026 年 6 月，Claude Code 和 Codex 已经不是简单的“代码插件”，而是由账号套餐、地区支持、用量限制、模型能力、账单和数据边界共同决定的生产力基础设施。

01

官方账号是默认入口

Claude Code 可通过 Claude Pro/Max 等订阅或 Console API 使用，Claude 与 Claude Code 的用量会共享；Codex 绑定 ChatGPT 账号与对应套餐，达到限制后可能需要购买额外 credits 或切到 API 计费。

02

中转站不是企业方案

灰色中转站能绕开网络、支付或地区限制，但会带来账号封禁、模型替换、日志外泄、代码和客户数据被二次利用、没有 SLA 和不可审计等风险，不能作为生产系统依赖。

03

国内算力要分层使用

国内落地要同时考虑国产模型、国内云 MaaS、私有化部署和海外官方 API：高价值复杂任务走合规强模型，高并发中文任务优先看国内算力，敏感数据优先本地或专有环境。

04

选型要看可持续性

不要只比较模型榜单。还要看账号所有权、团队权限、峰值并发、延迟、价格波动、上下文成本、数据留存、审计日志、备用模型和人工降级路径。

01

战略与场景

为什么重要
没有业务目标，AI 项目会变成工具试用，难以判断投入是否值得扩大。
准备什么
明确优先业务、目标指标、试点流程、责任人和可接受的收益周期。
检查问题
是否能说清这次 AI 试点要改善哪一个指标，以及不用 AI 时的当前基线？
02

数据与知识

为什么重要
AI 输出质量取决于材料质量，分散、过期或越权的数据会直接放大业务风险。
准备什么
梳理知识库、业务文档、数据来源、更新频率、权限分级和敏感信息边界。
检查问题
关键任务所需材料是否找得到、信得过、可授权，并且知道谁负责维护？
03

工具与平台

为什么重要
如果 AI 只停在个人聊天框或临时中转链路里，员工仍要复制粘贴，提效很难沉淀为团队流程。
准备什么
盘点办公套件、CRM、工单、文档、表格、知识库、AI 工具、企业账号、SSO/RBAC 和 API 集成入口。
检查问题
目标流程中的输入、输出、审批、追踪和账号权限，是否能在公司批准的工具里顺畅衔接？
04

算力与模型

为什么重要
模型能力、成本、延迟、地区可用性和国内算力稳定性会影响体验，也决定哪些场景适合规模化。
准备什么
明确模型选择、官方账号/API、国内云或私有部署、上下文长度、并发需求、预算上限和备用方案。
检查问题
高峰期是否有稳定供给，单位任务成本是否可接受，海外服务不可用或国内算力拥塞时是否有降级路径？
05

人才与流程

为什么重要
AI 改变的是工作分工，只有培训工具用法，不会自动形成新的协作方式。
准备什么
设计岗位 SOP、培训任务、提示词样例、人机分工、审核节点和复盘节奏。
检查问题
员工是否知道哪些任务交给 AI、哪些必须人工判断、结果不合格时找谁处理？
06

治理与评估

为什么重要
没有质量、安全和责任边界，AI 越深入流程，潜在错误和合规风险越难追踪。
准备什么
建立输出标准、抽检机制、日志记录、风险分级、审批规则和事故复盘模板。
检查问题
当 AI 输出错误、泄露敏感信息或影响客户时，是否知道如何发现、升级和纠正？

Market Map

工具、模型与 Agent 图谱：先认识代表性能力，再决定怎么接入

这些名字不是采购清单，而是帮助企业理解当下 AI 能力边界：哪些适合做通用模型供给，哪些适合进入研发流程，哪些适合内容生产，哪些适合沉淀成 Agent、Skills 和工作流，以及哪些必须先解决账号、合规访问和算力稳定性。

Compute & Models

算力与模型供给

代表工具
OpenAI GPT/Codex 系列、Claude、Gemini、Qwen、GLM/Z.ai、DeepSeek、Kimi。
适合场景
通用问答、中文知识工作、代码生成、长文档分析、多模态理解和企业 API 接入。
导入前准备
准备评测集、成本预算、调用限额、官方账号、国内云/私有化策略、延迟要求、备用模型和降级路径。
Coding Agents

代码与研发 Agent

代表工具
Codex、Claude Code、Qwen Code、GLM coding、Cursor、Windsurf Cascade、GitHub Copilot coding agent、Devin、Cline。
适合场景
需求拆解、代码生成、重构、测试补齐、PR 审查、文档生成、CI 失败定位和遗留系统理解。
导入前准备
准备代码权限、沙箱环境、测试命令、分支策略、审查规则、AGENTS.md/Rules、账号归属、用量监控、日志和人类批准点。
Creative & Video

内容与视频创作工具

代表工具
Seedance 2.0、Google Veo、OpenAI Sora、Runway Gen-4、可灵 Kling、MiniMax/Hailuo、Vidu、PixVerse、LiblibAI。
适合场景
短视频脚本、广告分镜、产品演示、培训素材、直播切片、品牌视觉探索和营销素材批量生产。
导入前准备
准备品牌规范、素材授权、人物肖像/IP 边界、审核流程、提示词模板、分镜标准和版权风险判断。
Agents & Skills

Agent、Skills 与工作流平台

代表工具
Claude Skills、Codex AGENTS.md/Skills、MCP、OpenAI Agents SDK、LangGraph、Google ADK、Dify、Coze、n8n、Flowise。
适合场景
把重复任务封装成可复用流程，连接知识库、表格、业务系统和审批节点，让 AI 能调用工具并留下轨迹。
导入前准备
准备工具权限、输入输出契约、异常升级、人机交接、运行日志、审批策略、沙箱测试和回滚方案。

Action

            
从明天开始的三件事

            
不要等“大模型战略”完整成型，先让一个真实流程跑起来。

          

          

            

              01
              
选一个流程

              
从高频、重复、可评价的工作开始，例如客户跟进、工单摘要、活动复盘、会议纪要。

            

            

              02
              
定账号和算力边界

              
明确用官方账号、企业 API、国内模型还是私有部署；中转站只作为风险样本，不进入客户数据、代码和生产流程。

            

            

              03
              
质检并复盘

              
明确输出格式、不可出错的边界、人工审核点和成功指标，记录节省时间、返工原因、使用阻力和业务结果。
